# Что это

Простой репортер о новых issue в репозитории. Присылает уведомление в Telegram, если создан новый issue.

Планировалось использование _в проектном практикуме_.

Уведомление будет отправлено если:

- у issue есть ответственные (исполнители)
- у issue автор и ответственный не совпадают - смысла о таком уведомлять нет

Пример уведомления:

```text
yakovlev05 создал задачу "github-project-reporter #3 Сборка на native image" и назначил ответственного - username
```

# Запуск

Запуск локально:

```shell
docker compose up -d
```

файл `docker-compose-prod.yaml` используется для деплоя на прод.

# Переменные окружения

Тут только описание тех переменных окружения, которые могут быть не понятны

- `GITHUB_TOKEN` - токен от гитхаба, обязательно право на **repo**
- `NEW_ISSUES_REPORTER_TASK_DELAY` - время между проверками новых issue, в секундах
- `TIME_WINDOW_TO_NOTIFY` - насколько старым по дате создания может быть issue, чтобы о нем уведомить, в сеекундах
- `TELEGRAM_BOT_TOKEN` - токен от бота в телеграм
- `TELEGRAM_CHAT_ID` - chat id для отправки сообщений
- `APP_IS_PRODUCTION` - false включает показ sql запросов в консоли
